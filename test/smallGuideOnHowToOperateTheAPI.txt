the api route /dropBox/fileDownload with query parameters such as the path from where the download should start and access_token that will be generated by you from the dropbox.

the api route /dropBox/fileList will provide you the folder structure from the root of the drop box access token must be provided.

all the data downloaded from the api will be provided in the test folder with time and date of the download and
inside it the structure will be same as in the drop box.

for running the api port should be provided in env
PORT=""

for using the api without providing the key the following env should be provided.
CLIENT_ID=""
CLIENT_SECRET=""
REFRESH_TOKEN=""
